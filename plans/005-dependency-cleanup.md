## Why this matters

`apps/web/package.json` declares 13 dependencies that are imported nowhere in the
app â€” leftovers from a HeroUI AI-chat scaffold export (markdown rendering, charts,
carousels, code highlighting). They bloat installs, slow CI, enlarge the
dependency-audit surface, and make it ambiguous which deps are actually load-
bearing. Separately, the monorepo has two version skews: `apps/mobile` pins
TypeScript `~6.0.3` while the root and web use `^5.7.2`, and `@supabase/supabase-js`
is `^2.106.2` in web but `^2.47.10` in `packages/api`. Both invite subtle
"works in one workspace, breaks in another" bugs. This is low-risk hygiene that
makes the next person's (and the next agent's) mental model of the repo accurate.
